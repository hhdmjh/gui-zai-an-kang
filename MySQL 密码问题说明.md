# ⚠️ MySQL 密码问题说明

## 问题现状

经过测试，当前系统的 MySQL 密码配置不正确，导致无法连接数据库。

**已尝试的密码：**
- ❌ `root`
- ❌ `123456`  
- ❌ 空密码
- ❌ `password`

**错误信息：**
```
ERROR 1045 (28000): Access denied for user 'root'@'localhost'
```

---

## 🔧 解决方案（3 选 1）

### 方案 1：找到正确的 MySQL 密码（推荐）

**回忆一下：**
1. 你安装 MySQL 8.0 时设置的密码是什么？
2. 是否保存在某个密码管理器中？
3. 是否告诉过别人？

**找到密码后：**
1. 打开 `D:\桂在安康\backend\.env`
2. 修改 `DB_PASSWORD=你的密码`
3. 运行 `npm run init-db`
4. 运行 `npm start`

---

### 方案 2：重置 MySQL 密码（简单）

**使用 MySQL 官方重置工具：**

1. 停止 MySQL 服务：
```bash
net stop MySQL80
```

2. 以安全模式启动（跳过密码验证）：
```bash
mysqld --skip-grant-tables --skip-networking
```

3. 打开新的命令行窗口，重置密码：
```bash
mysql -u root
```

4. 在 MySQL 命令行执行：
```sql
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED BY '123456';
FLUSH PRIVILEGES;
EXIT;
```

5. 重启 MySQL：
```bash
net start MySQL80
```

6. 修改 `.env` 文件：
```env
DB_PASSWORD=123456
```

---

### 方案 3：安装 XAMPP（最简单，无需配置）

**步骤：**

1. 下载 XAMPP：
   - 访问：https://www.apachefriends.org/download.html
   - 下载 Windows 版本
   - 安装到 `C:\xampp`

2. 启动 XAMPP：
   - 打开 XAMPP Control Panel
   - 点击 MySQL 的 "Start" 按钮

3. 修改 `.env` 文件：
```env
DB_PASSWORD=
```
（等号后面什么都不写）

4. 运行：
```bash
cd D:\桂在安康\backend
npm run init-db
npm start
```
