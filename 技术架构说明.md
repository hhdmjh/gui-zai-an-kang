## 🏗️ 架构对比

### 当前架构（纯前端）

```
┌─────────────────────────┐
│   浏览器 (前端)          │
│  ┌───────────────────┐  │
│  │  HTML/CSS/JS      │  │
│  ├───────────────────┤  │
│  │  ECharts 图表     │  │
│  ├───────────────────┤  │
│  │  LocalStorage     │  │
│  │  (数据存储在本地)  │  │
│  └───────────────────┘  │
└─────────────────────────┘
```

**优点：**
- ✅ 无需安装，打开浏览器即可使用
- ✅ 部署简单，只需静态文件
- ✅ 无服务器成本

**缺点：**
- ❌ 数据只在本地，换设备数据丢失
- ❌ 无法多设备同步
- ❌ 无用户管理
- ❌ 数据可能丢失（清除浏览器数据）

### 升级后架构（前后端分离）

```
┌─────────────────────────┐      HTTP/JSON     ┌─────────────────────────┐
│   浏览器 (前端)          │ ◄──────────────►  │   后端服务器             │
│  ┌───────────────────┐  │                   │  ┌───────────────────┐   │
│  │  HTML/CSS/JS      │  │                   │  │  Node.js + Express │   │
│  ├───────────────────┤  │                   │  ├───────────────────┤   │
│  │  ECharts 图表     │  │                   │  │  REST API         │   │
│  ├───────────────────┤  │                   │  ├───────────────────┤   │
│  │  LocalStorage     │  │                   │  │  业务逻辑         │   │
│  │  (缓存)           │  │                   │  └─────────┬─────────┘   │
│  └───────────────────┘  │                   │            │            │
└─────────────────────────┘                   │            ▼            │
                                              │  ┌───────────────────┐   │
                                              │  │  MySQL 数据库     │   │
                                              │  │  (持久化存储)     │   │
                                              │  └───────────────────┘   │
                                              └─────────────────────────┘
```

**优点：**
- ✅ 数据持久化存储，不会丢失
- ✅ 多设备同步，随时随地访问
- ✅ 完整的用户管理系统
- ✅ 数据安全可靠
- ✅ 支持多用户
- ✅ 可扩展性强

**缺点：**
- ❌ 需要安装 Node.js 和 MySQL
- ❌ 需要服务器运行后端服务
- ❌ 部署相对复杂

## 🔌 API 接口说明

### 已实现的接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/login` | POST | 用户登录 |
| `/api/health-data` | GET | 获取健康数据 |
| `/api/health-data` | POST | 保存健康数据 |
| `/api/health-data/:id` | DELETE | 删除健康数据 |
| `/api/appointments` | GET | 获取预约记录 |
| `/api/appointments` | POST | 创建预约 |

### 数据表结构

**users（用户表）**
- id: 用户 ID
- username: 用户名
- password: 密码
- created_at: 创建时间

**health_data（健康数据表）**
- id: 记录 ID
- user_id: 用户 ID
- record_date: 记录日期
- systolic: 收缩压
- diastolic: 舒张压
- blood_sugar: 血糖
- sleep: 睡眠时长
- weight: 体重
- sport: 运动时长
- created_at: 创建时间
- updated_at: 更新时间

**appointments（预约记录表）**
- id: 预约 ID
- user_id: 用户 ID
- name: 姓名
- phone: 电话
- appointment_date: 预约日期
- appointment_time: 预约时间
- address: 地址
- remark: 备注
- status: 状态
- created_at: 创建时间

## 🚀 升级步骤

### 1. 安装环境
```bash
# 安装 Node.js
winget install OpenJS.NodeJS.LTS

# 安装 MySQL（手动）
# 访问 https://dev.mysql.com/downloads/mysql/
```

### 2. 配置项目
```bash
cd backend
copy .env.example .env
# 编辑 .env 文件，配置数据库连接
npm install
```

### 3. 初始化数据库
```bash
npm run init-db
```

### 4. 启动服务
```bash
npm start
```

### 5. 前端集成
在 HTML 中添加：
```html
<script src="backend/api-client.js"></script>
```

修改数据操作：
```javascript
// 原来
localStorage.setItem('healthData', JSON.stringify(data));

// 现在
await SyncManager.saveHealthData(data);
```
