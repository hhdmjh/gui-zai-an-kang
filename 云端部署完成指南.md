## 第 1 步：注册 GitHub 账号（如果没有）

**访问：** https://github.com/signup

1. 输入邮箱地址
2. 设置密码
3. 设置用户名（记住这个用户名！）
4. 完成验证

**时间：** 2 分钟

---

## 第 2 步：在 GitHub 创建仓库

**访问：** https://github.com/new

1. **Repository name** 填写：`gui-zai-an-kang`
2. 选择 **Public**（公开）
3. 点击 **Create repository**

**时间：** 1 分钟

---

## 第 3 步：推送代码到 GitHub

在你的项目文件夹打开命令行（PowerShell 或 CMD），执行：

```bash
cd d:\桂在安康

# 设置远程仓库（替换为你的 GitHub 用户名）
git remote add origin https://github.com/你的用户名/gui-zai-an-kang.git

# 设置主分支
git branch -M main

# 推送代码
git push -u origin main
```

**如果提示输入密码：**
- 使用你的 GitHub 密码
- 或者使用 Personal Access Token（推荐）

**获取 Personal Access Token：**
1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token"
3. 选择权限（至少选 repo）
4. 生成后复制 token
5. 用 token 作为密码

**时间：** 2 分钟

---

## 第 4 步：部署前端到 Vercel

**访问：** https://vercel.com

1. 点击 "Sign up" 或 "Login"
2. 选择 "Continue with GitHub"（用 GitHub 登录）
3. 登录后，点击 "Import Project"
4. 找到 `gui-zai-an-kang` 项目
5. 点击 "Import"
6. 点击 "Deploy"
7. 等待 1-2 分钟

**部署成功后，你会得到：**
```
https://gui-zai-an-kang.vercel.app
```

**时间：** 3 分钟

---

## 第 5 步：部署后端到 Railway

**访问：** https://railway.app

1. 点击 "Login" 或 "Sign up"
2. 选择 "Login with GitHub"
3. 登录后，点击 "New Project"
4. 选择 "Deploy from GitHub repo"
5. 选择 `gui-zai-an-kang` 项目
6. Railway 会自动检测并部署
7. 等待 2-3 分钟

**部署成功后，你会得到后端地址：**
```
https://gui-zai-an-kang-production.up.railway.app
```

**时间：** 3 分钟

---

## 第 6 步：配置 API 地址

现在需要告诉前端后端的地址。

### 修改文件：
修改以下 6 个文件中的 API 地址：

1. `home.html`
2. `login.html`
3. `register.html`
4. `profile.html`
5. `notifications.html`
6. `contacts.html`

### 修改内容：

**找到这行代码：**
```javascript
const API_BASE_URL = 'http://localhost:3000/api';
```

**改为：**
```javascript
const API_BASE_URL = 'https://你的后端地址.up.railway.app/api';
```

例如：
```javascript
const API_BASE_URL = 'https://gui-zai-an-kang-production.up.railway.app/api';
```

### 重新部署：

修改完成后，在命令行执行：

```bash
cd d:\桂在安康
git add .
git commit -m "更新 API 地址"
git push
```

然后在 Vercel 点击 "Redeploy" 重新部署前端。

**时间：** 5 分钟

## 📞 遇到问题？

### 常见错误：

**1. Git push 失败**
- 检查 GitHub 用户名是否正确
- 使用 Personal Access Token 代替密码

**2. Vercel 部署失败**
- 确保 GitHub 仓库是 Public
- 检查是否选择了正确的仓库

**3. Railway 部署失败**
- 检查 Procfile 是否存在
- 查看 Deploy Logs 找错误

**4. 前端无法连接后端**
- 检查 API 地址是否正确
- 测试后端：`https://你的地址.up.railway.app/api/health`
