# 🔧 问题解决指南

## 问题：数据库连接失败

**错误信息：** `Access denied for user 'root'@'localhost' (using password: YES)`

**原因：** MySQL 密码配置错误

---

## ✅ 解决方案（3 步）

### 第 1 步：找到你的 MySQL 密码

**回想一下：**
- 你安装 MySQL 时设置的 root 密码是什么？
- 如果用的是 XAMPP，默认密码通常是**空**（没有密码）
- 如果改过密码，就是你改的那个

**常见密码：**
- `root`
- `123456`
- `password`
- 空（直接回车）

---

### 第 2 步：修改配置文件

打开文件：`D:\桂在安康\backend\.env`

找到这一行：
```env
DB_PASSWORD=root
```

改成你的实际密码，比如：
```env
DB_PASSWORD=123456
```

**如果是 XAMPP（密码为空）：**
```env
DB_PASSWORD=
```

---

### 第 3 步：重新启动

修改后，在命令行执行：

```bash
cd D:\桂在安康\backend
npm run init-db
npm start
```

看到以下提示表示成功：
```
╔════════════════════════════════════════════════════════╗
║          🌿 桂在安康 - 后端服务启动成功                ║
╚════════════════════════════════════════════════════════╝
```

然后浏览器访问：`http://localhost:3000`

---

## 🔍 如果不知道 MySQL 密码怎么办？

### 方法 1：使用 XAMPP
如果你用的是 XAMPP：
1. 打开 XAMPP Control Panel
2. 启动 MySQL
3. 点击"Admin"打开 phpMyAdmin
4. 默认不需要密码

修改 `.env` 为：
```env
DB_PASSWORD=
```

### 方法 2：重置 MySQL 密码（Windows）

1. 停止 MySQL 服务
2. 创建文本文件 `reset.txt`，内容：
```sql
ALTER USER 'root'@'localhost' IDENTIFIED BY '123456';
```
3. 用这个命令启动 MySQL：
```bash
mysqld --init-file=C:\\path\\to\\reset.txt
```
4. 密码就重置为 `123456` 了

## 📞 常见问题

### Q: 我怎么知道 MySQL 密码是多少？
**A:** 
- 如果是你自己安装的，回想一下安装时设置的密码
- 如果是别人帮你装的，问那个人
- 如果用的是 XAMPP/ApacheFriends，通常没有密码（空）

### Q: 我试了所有密码都不行怎么办？
**A:** 重置 MySQL 密码（参考上面的方法 2）

### Q: 服务器启动了，但浏览器打不开？
**A:** 
1. 确认地址是 `http://localhost:3000`（不是 3001 或其他）
2. 检查服务器是否在运行（命令行窗口应该显示"服务启动成功"）
3. 换个浏览器试试
