## 🎯 方案一：本地演示（最简单，适合现场演示）

### 适用场景：
- ✅ 比赛现场演示
- ✅ 给评委展示
- ✅ 给朋友演示
- ✅ 不需要联网

### 操作步骤：

#### 1. 启动服务器
在你的电脑上双击运行：
```
一键启动服务器.bat
```

#### 2. 打开浏览器
访问：`http://localhost:3000`

#### 3. 演示流程
1. **注册账号** - 现场注册一个账号
2. **登录系统** - 用刚注册的账号登录
3. **录入数据** - 录入一些健康数据
4. **查看图表** - 展示数据趋势图
5. **AI 报告** - 生成 AI 健康分析
6. **预警推送** - 演示发送预警给联系人
7. **消息中心** - 展示接收到的预警

### 优点：
- ✅ 简单快速
- ✅ 不需要网络
- ✅ 完全控制

### 缺点：
- ❌ 只能在你的电脑上用
- ❌ 别人不能远程访问

## 🌐 方案二：局域网访问（适合小范围使用）

### 适用场景：
- ✅ 同一 WiFi 下的多设备
- ✅ 教室/办公室内演示
- ✅ 手机、平板都能访问

### 操作步骤：

#### 1. 启动服务器
双击运行：`一键启动服务器.bat`

#### 2. 查看你的 IP 地址
在电脑上：
1. 按 `Win + R`
2. 输入 `cmd` 回车
3. 输入 `ipconfig` 回车
4. 找到 `IPv4 地址`，例如：`192.168.1.100`

#### 3. 让别人访问
告诉别人访问地址：
```
http://你的 IP 地址：3000
```
例如：`http://192.168.1.100:3000`

#### 4. 手机也能访问
手机连接同一 WiFi，浏览器输入：
```
http://192.168.1.100:3000
```

### 优点：
- ✅ 多设备同时访问
- ✅ 手机、平板都能用
- ✅ 不需要额外配置

### 缺点：
- ❌ 必须在同一 WiFi 下
- ❌ 离开局域网就无法访问

## ☁️ 方案三：云端部署（最佳，适合全球访问）

### 适用场景：
- ✅ 比赛作品提交
- ✅ 让评委回家也能访问
- ✅ 全球各地都能用
- ✅ 7×24 小时在线

### 部署步骤：

#### 第 1 步：准备 GitHub 账号
1. 访问：https://github.com/signup
2. 注册账号（免费）
3. 记住用户名

#### 第 2 步：推送代码到 GitHub
在你的项目文件夹打开命令行，执行：

```bash
# 初始化 Git（如果还没初始化）
git init

# 添加所有文件
git add .

# 提交
git commit -m "桂在安康参赛作品"

# 设置分支名
git branch -M main

# 添加远程仓库（替换为你的用户名）
git remote add origin https://github.com/你的用户名/gui-zai-an-kang.git

# 推送到 GitHub
git push -u origin main
```

#### 第 3 步：部署到 Vercel（前端）
1. 访问：https://vercel.com
2. 用 GitHub 账号登录
3. 点击 "Import Project"
4. 选择你的项目 `gui-zai-an-kang`
5. 点击 "Deploy"
6. 等待 1-2 分钟

部署成功后，你会得到一个网址：
```
https://gui-zai-an-kang.vercel.app
```

#### 第 4 步：部署后端到 Railway
1. 访问：https://railway.app
2. 用 GitHub 账号登录
3. 点击 "New Project"
4. 选择 "Deploy from GitHub repo"
5. 选择你的项目
6. 等待自动部署

部署成功后，你会得到后端地址：
```
https://gui-zai-an-kang-production.up.railway.app
```

#### 第 5 步：配置前端 API 地址
修改 `home.html`、`login.html`、`register.html` 等文件中的 API 地址：

```javascript
// 原来的
const API_BASE_URL = 'http://localhost:3000/api';

// 改为
const API_BASE_URL = 'https://你的后端地址.up.railway.app/api';
```

然后重新部署到 Vercel。

### 优点：
- ✅ 全球都能访问
- ✅ 不需要你的电脑开机
- ✅ 7×24 小时在线
- ✅ 专业、正式

### 缺点：
- ❌ 需要配置
- ❌ Railway 免费额度有限

## 🔧 常见问题

### Q1: 服务器启动失败？
**解决：**
1. 检查是否安装了 Node.js
2. 检查端口 3000 是否被占用
3. 重启电脑后重试

### Q2: 别人访问不了？
**解决：**
1. 检查防火墙设置
2. 确保在同一网络
3. 检查 IP 地址是否正确

### Q3: 云端部署太复杂？
**解决：**
1. 先用本地演示
2. 有时间再研究云端部署
3. 可以寻求技术支持
