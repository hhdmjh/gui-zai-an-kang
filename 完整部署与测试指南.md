## 📊 数据库表结构

### ai_health_reports 表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 用户 ID（外键） |
| report_date | DATE | 报告日期 |
| overall_score | INT | 综合评分（0-100） |
| bp_score | INT | 血压评分（0-100） |
| sugar_score | INT | 血糖评分（0-100） |
| lifestyle_score | INT | 生活方式评分（0-100） |
| avg_systolic | DECIMAL | 平均收缩压 |
| avg_diastolic | DECIMAL | 平均舒张压 |
| avg_blood_sugar | DECIMAL | 平均血糖 |
| avg_sleep | DECIMAL | 平均睡眠 |
| avg_weight | DECIMAL | 平均体重 |
| avg_sport | INT | 平均运动 |
| suggestions | TEXT | 健康建议（JSON） |
| warning_suggestions | TEXT | 预警建议（JSON） |
| trend_analysis | TEXT | 趋势分析（JSON） |
| created_at | TIMESTAMP | 创建时间 |

## 🎯 功能说明

### 1. 重置表单按钮

**作用**：清空当前输入的表单数据

**使用场景**：
- 填错了，想重新填写
- 测试不同数据
- 取消当前输入

**效果**：
- ✅ 清空所有输入框
- ✅ 日期回到今天
- ✅ 不影响已提交的数据

---

### 2. AI 健康建议

**作用**：基于用户的历史健康数据，生成个性化的健康建议

**算法逻辑**：

1. **数据收集**
   - 获取用户所有健康数据记录
   - 计算各项指标的平均值
   - 分析数据趋势

2. **评分系统**
   - 血压评分：基于收缩压和舒张压
   - 血糖评分：基于空腹血糖
   - 生活方式评分：基于睡眠、运动、体重

3. **建议生成**
   - 血压偏高 → 低盐饮食建议
   - 血糖偏高 → 控制碳水建议
   - 睡眠不足 → 改善睡眠建议
   - 运动不足 → 增加运动建议
   - 体重上升 → 体重管理建议

4. **预警提示**
   - 血压≥140/90 → 高血压预警
   - 血糖≥7.0 → 糖尿病预警
   - 睡眠<5 小时 → 严重睡眠不足预警

**使用建议**：
- 至少录入 3 条数据再查看 AI 建议（更准确）
- 每天坚持录入数据
- 定期查看 AI 建议，调整生活习惯

## 🔍 常见问题

### Q1: 重置按钮在哪里？
**A**: 在首页的"健康数据录入"表单底部，和"提交数据"按钮并排。

### Q2: AI 建议准不准？
**A**: AI 建议基于医学标准和您的历史数据生成，具有参考价值，但不能替代医生诊断。

### Q3: 为什么 AI 建议页面显示空白？
**A**: 可能原因：
- 还没有健康数据记录 → 先去录入数据
- 数据太少 → 建议至少录入 3 条
- 后端服务未启动 → 检查 server.js 是否运行

### Q4: 数据库更新后原来的数据会丢失吗？
**A**: 不会！更新脚本只是添加新表，不会影响已有数据。

### Q5: AI 建议多久生成一次？
**A**: 每次点击"获取 AI 健康建议"都会实时生成，基于最新的数据。
