# 🚀 快速部署后端到 Railway

## 为什么需要部署后端？
GitHub Pages 只能托管前端页面，无法运行 Node.js 后端。
要让线上版本正常工作，需要将后端部署到云端。

---

## 方案 A：使用 Railway（推荐，最简单）⭐

### 步骤 1：访问 Railway
👉 打开：https://railway.app/

### 步骤 2：创建账号
1. 点击 "Login" 或 "Start a New Project"
2. 使用 GitHub 账号登录

### 步骤 3：创建新项目
1. 点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 选择您的仓库：`gui-zai-an-kang`

### 步骤 4：配置后端
1. Railway 会自动识别 `backend` 文件夹
2. 点击 "backend" 服务
3. 点击 "Variables" 标签
4. 添加以下环境变量：

```
PORT=3000
DB_HOST=<您的 MySQL 主机>
DB_USER=<数据库用户名>
DB_PASSWORD=<数据库密码>
DB_NAME=gui_zai_an_kang
JWT_SECRET=gui_zai_an_kang_secret_key_2026
```

### 步骤 5：部署
1. Railway 会自动部署
2. 等待 2-3 分钟
3. 获得公网 URL，如：`https://gui-zai-an-kang-production.up.railway.app`

### 步骤 6：更新前端 API 地址
1. 回到 GitHub 仓库
2. 编辑所有包含 `API_BASE_URL` 的文件
3. 将 `http://localhost:3000` 改为 Railway 的 URL
4. 提交更改

---

## 方案 B：使用 Vercel 部署后端（备选）

Vercel 主要托管前端，但也可以通过 Serverless Functions 运行后端 API。

1. 将 `backend` 文件夹重命名为 `api`
2. 在 Vercel 部署
3. 获得 API URL

---

## 方案 C：使用 Render.com（免费备选）

1. 访问：https://render.com/
2. 创建账号
3. 创建 "Web Service"
4. 连接 GitHub 仓库
5. 配置环境变量
6. 部署
