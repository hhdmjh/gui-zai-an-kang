### 项目介绍（1 分钟）

**首先，让我介绍一下项目的核心功能。**

桂在安康是一个基于 **Node.js + MySQL** 的全栈应用，主要包含以下模块：

1. **健康数据管理** - 用户可以记录血压、血糖、睡眠、体重等多项指标
2. **智能预警系统** - 自动检测异常数据，及时发出健康预警
3. **AI 健康分析** - 基于数据生成个性化健康建议报告
4. **社区服务** - 提供上门体检预约、健康咨询等服务
5. **健康知识库** - 提供疾病护理、饮食养生等知识

**技术架构方面：**
- 后端采用 **Express 框架**，提供 RESTful API
- 数据库使用 **MySQL**，保证数据安全和一致性
- 前端采用**原生 JavaScript + ECharts**，实现数据可视化
- 使用 **JWT Token** 进行身份验证，**bcrypt** 加密密码

## 三、评委可能问的问题及回答

### Q1: 你的项目和市面上的健康 APP 有什么区别？

**回答：**
"谢谢老师的提问。我们的项目有三个独特之处：

1. **专注老年群体** - 针对老年人设计了大字体、简洁界面、一键操作
2. **社区服务对接** - 不仅记录数据，还连接社区医疗服务
3. **智能预警系统** - 异常数据会自动通知家属和医生，这是很多 APP 没有的

我们的定位是**社区健康管理的助手**，而不是简单的数据记录工具。"

---

### Q2: 你的 AI 分析是怎么实现的？

**回答：**
"我的 AI 分析主要包含三个部分：

1. **健康评分算法** - 根据医学标准，为各项指标打分，然后加权平均
2. **趋势分析** - 使用线性回归算法，判断数据是上升、下降还是稳定
3. **建议生成** - 基于规则和阈值，匹配相应的健康建议

比如血压评分，我会计算平均值与标准值（120/80）的偏差，偏差越大分数越低。

目前是基于规则引擎，未来计划引入机器学习模型，提高建议的准确性。"

---

### Q3: 数据安全性如何保证？

**回答：**
"我在数据安全方面做了以下措施：

1. **密码加密** - 使用 bcrypt 加密，即使数据库泄露也无法还原密码
2. **Token 认证** - 每次请求都需要验证 JWT Token，防止未授权访问
3. **SQL 注入防护** - 使用参数化查询，避免 SQL 注入
4. **数据隔离** - 每个用户只能访问自己的数据
5. **HTTPS** - 生产环境会使用 HTTPS 加密传输

这些都是企业级的安全标准。"

---

### Q4: 如果同时有很多人使用，系统能承载吗？

**回答：**
"目前我使用的是单服务器架构，对于小规模用户（几百人）是可以的。

如果要支持大规模并发，我会：
1. 使用**连接池**管理数据库连接
2. 添加**Redis 缓存**，减少数据库查询
3. 使用**负载均衡**，部署多台服务器
4. 对数据库进行**读写分离**

这些是我下一步要优化的内容。"

---

### Q5: 你的预警准确吗？误报怎么办？

**回答：**
"预警阈值我是参考**中国高血压防治指南**和**糖尿病诊断标准**设定的，比如：
- 收缩压 ≥ 140mmHg → 高血压
- 血糖 ≥ 7.0mmol/L → 糖尿病

这个标准是医学界公认的，准确性有保障。

对于误报，我有两个解决方案：
1. **多次测量确认** - 建议用户复测，避免偶然误差
2. **人工审核** - 社区医生可以手动确认或取消预警

未来我会引入更智能的算法，考虑个体差异，提高准确性。"
